摘要选择的优先顺序
由于摘要的表达方式多种多样，了解其顺序非常重要。顺序如下：

如果<!--more-->文章中存在摘要分隔符，但分隔符之前没有任何内容，则描述将用作摘要。
如果<!--more-->文章中存在摘要分隔符，则分隔符之前的文本将按照手动摘要拆分方法提供。
如果文章前置元数据中存在摘要变量，则该变量的值将按照前置元数据摘要方法提供。
文章开头的文本将按照自动摘要拆分方式提供。

引用图像和音乐等本地资源主要有三种方式：

使用“页面包”（page bundles）中的页面资源。你可以通过 `Resources.GetMatch` 的匹配值，或者直接使用相对于页面目录的资源文件路径来引用这些资源。
将资源存储在 `assets` 目录中（默认为 `/assets`）。在文章中引用时，资源路径需相对于 `assets` 目录。
将资源存储在 `static` 目录中（默认为 `/static``）。在文章中引用时，资源路径需相对于 `static` 目录。
上述引用方式的优先级也遵循此顺序。

主题中许多地方都可以使用上述本地资源引用方式，例如链接、图像、图像短代码（shortcode）、音乐短代码以及 Front Matter 中的某些参数。

未来将支持对页面资源或 `assets` 目录中的图像进行处理。这真的很棒！

There are three ways to reference local resources such as images and music:

Using page resources in page bundles. You can reference page resources by the value for Resources.GetMatch or the filepath of the resource relative to the page directory directly.
Store resources in the assets directory, which is /assets by default. The filepath of the resource to reference in the post is relative to the assets directory.
Store resources in the static directory, which is /static by default. The filepath of the resource to reference in the post is relative to the static directory.
The priority of references is also in the above order.

There are many places in the theme where the above local resource references can be used, such as links, images, image shortcode, music shortcode and some params in the front matter.

Images in page resources or assets directory processing will be supported in the future. It’s really cool!
